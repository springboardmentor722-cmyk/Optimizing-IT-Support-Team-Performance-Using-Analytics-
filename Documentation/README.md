## Optimizing IT Support Team Performance Using Analytics ( Supportlytics)


## Project Overview

This project analyzes IT support ticket data to identify performance trends, optimize resolution times, and enhance service efficiency using data analytics and visualization techniques.

The goal is to uncover patterns in ticket requests, technical issues, and support performance metrics to improve workflow and resource allocation.

---

## Project Workflow

The project follows a structured analytics pipeline:

1. Data Acquisition
2. Data Cleaning
3. Feature Engineering
4. Exploratory Data Analysis
5. Cluster and Similarity Analysis
6. Performance Trend Analysis
7. Geographic Insights
8. Dashboard Development
9. Insights and Recommendations

---

## Project Milestones

**Milestone 1 - Data Acquisition and Understanding**
- Dataset loading and inspection
- Schema and data type exploration
- Ticket distribution analysis by Type, Priority, and Category

**Milestone 2 - Data Cleaning and Feature Engineering**
- Missing value handling
- Feature engineering: Resolution Duration, Priority Score
- Exploratory data analysis and pattern discovery

**Milestone 3 - Cluster and Performance Analysis**
- Cluster creation and similarity score calculation
- Performance trend analysis across priorities and regions
- Geographic insights using heatmaps and geo-visualizations

**Milestone 4 - Dashboard and Documentation**
- Power BI dashboard with key performance KPIs
- Streamlit interactive dashboard
- Final project report and presentation

---

## Tech Stack

**Data Processing**
- Python, Pandas, NumPy

**Visualization**
- Matplotlib, Seaborn, Plotly

**Dashboard**
- Power BI, Streamlit

---

## Key Metrics

- **Resolution Time** - Time taken to resolve support tickets
- **Response Efficiency** - Efficiency metric based on resolution speed
- **Priority Distribution** - Distribution of tickets across priority levels
- **Similarity Score** - Metric used to identify similarity between issues
- **Cluster Performance** - Performance comparison across issue clusters
