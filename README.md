# IT Support Performance Analytics Dashboard (Supportlytics)

## 1. Project Overview

This project focuses on analyzing IT support ticket data to identify performance trends, optimize resolution time, and improve service efficiency using data analytics and visualization techniques.

The objective is to uncover patterns in ticket requests, issue categories, and support performance metrics to recommend improvements in workflow and resource allocation.

---

## 2. Objectives

* Analyze IT support ticket dataset
* Identify trends in ticket volume, priority, and resolution time
* Perform clustering and similarity analysis on issues
* Evaluate support team performance
* Build interactive dashboards for visualization
* Generate actionable insights and recommendations

---

## 3. Project Workflow

The project follows a structured analytics pipeline:

1. Data Acquisition & Understanding
2. Data Cleaning & Preprocessing
3. Feature Engineering
4. Exploratory Data Analysis (EDA)
5. Cluster & Similarity Analysis
6. Performance Trend Analysis
7. Geographic Insights
8. Dashboard Development
9. Insights & Recommendations

---

## 4. Project Milestones

### 🔹 Milestone 1 (Week 1–2): Data Preparation

* Dataset loading and inspection
* Handling missing values
* Feature engineering (Resolution Duration, Priority Score)
* Initial analysis of ticket distribution

### 🔹 Milestone 2 (Week 3–4): Exploratory Analysis

* Visualization of ticket types and priorities
* Identification of top issue categories
* Cluster and similarity insights
* Minimum 8 visualizations generated

### 🔹 Milestone 3 (Week 5–6): Performance Analysis

* Resolution time analysis by priority and type
* Identification of high-performing regions/teams
* Detection of unresolved high-priority issues

### 🔹 Milestone 4 (Week 7–8): Dashboard & Reporting

* Power BI dashboard creation
* KPI-based visualization (Resolution Time, Categories, Clusters)
* Final report and presentation
* GitHub project deployment

## 6. Key Metrics

* **Resolution Time** – Time taken to resolve tickets
* **Response Efficiency** – Efficiency based on resolution speed
* **Priority Distribution** – Ticket distribution by priority levels
* **Similarity Score** – Measures similarity between issues
* **Cluster Performance** – Performance of grouped issues
